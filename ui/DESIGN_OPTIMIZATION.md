# WorkAgent 视觉优化方案
## 资深 UI/UX 设计师视角的现代 SaaS 界面重塑

---

## 1. 问题诊断

### 当前界面视觉问题清单

| 问题类别 | 具体表现 | 视觉影响 |
|---------|---------|---------|
| **线条过重** | 边框使用 1px solid，颜色对比度过高 | 产生"线稿感"，模块间生硬割裂 |
| **缺乏层次** | 背景色阶跳跃过大（白→灰→深灰） | 视觉断层，缺乏通透融合感 |
| **圆角不统一** | 卡片、按钮、输入框圆角各异 | 整体感缺失，显得随意 |
| **阴影粗糙** | 阴影范围大、模糊度低、颜色偏灰 | 产生"脏"的感觉，缺乏高级感 |
| **色彩噪点** | 品牌色饱和度过高，辅助色过多 | 视觉疲劳，焦点分散 |
| **间距混乱** | 内边距、外边距无规律 | 呼吸感不足，信息密度不均 |
| **交互反馈弱** | 悬停态仅改变背景色 | 缺乏精致微交互，体验平淡 |

### 核心矛盾
当前设计停留在"功能可用"层面，缺乏"情感共鸣"。用户面对的是一个工具，而非一个"懂设计"的 AI 伙伴。

## 2. 设计系统规范

### 2.1 色彩系统（去噪化重构）

#### 主色（品牌色）
```
Primary: #6366F1 → #7C7EF6（降低饱和度，提升高级感）
Primary Light: rgba(124, 126, 246, 0.08)（用于背景填充）
Primary Glow: rgba(124, 126, 246, 0.25)（用于发光效果）
```

#### 中性色（透明度层级）
```
背景层级：
- Surface: #FFFFFF（主背景）
- Elevated: rgba(0, 0, 0, 0.02)（一级提升，卡片）
- Overlay: rgba(0, 0, 0, 0.04)（二级提升，选中态）
- Pressed: rgba(0, 0, 0, 0.06)（按下态）

文字层级：
- Primary: rgba(0, 0, 0, 0.88)（标题）
- Secondary: rgba(0, 0, 0, 0.56)（正文）
- Tertiary: rgba(0, 0, 0, 0.36)（辅助信息）
- Disabled: rgba(0, 0, 0, 0.22)（禁用态）
```

#### 功能色（降低侵略性）
```
Success: #10B981 → #34D399（更柔和的绿色）
Warning: #F59E0B → #FBBF24（暖黄而非橙黄）
Error: #EF4444 → #F87171（降低红色侵略性）
```

### 2.2 圆角系统（统一阶梯）

| 层级 | 圆角值 | 使用场景 |
|-----|-------|---------|
| XS | 6px | 标签、小按钮 |
| SM | 8px | 输入框、小卡片 |
| MD | 12px | 标准卡片、按钮 |
| LG | 16px | 大卡片、模态框 |
| XL | 20px | 特殊强调卡片 |
| FULL | 9999px | 胶囊按钮、头像 |

**核心规则**：同一界面内，圆角差异不超过 2 个阶梯。

### 2.3 阴影系统（柔和层级）

```css
/* Level 1: 微悬浮（按钮悬停） */
shadow-sm: 0 1px 2px rgba(0, 0, 0, 0.04), 0 1px 3px rgba(0, 0, 0, 0.02)

/* Level 2: 卡片默认 */
shadow-md: 0 4px 6px -1px rgba(0, 0, 0, 0.03), 0 2px 4px -2px rgba(0, 0, 0, 0.02)

/* Level 3: 卡片悬停/下拉菜单 */
shadow-lg: 0 10px 15px -3px rgba(0, 0, 0, 0.05), 0 4px 6px -4px rgba(0, 0, 0, 0.02)

/* Level 4: 模态框/弹窗 */
shadow-xl: 0 20px 25px -5px rgba(0, 0, 0, 0.06), 0 8px 10px -6px rgba(0, 0, 0, 0.02)

/* 特殊：品牌发光 */
glow-primary: 0 0 20px rgba(124, 126, 246, 0.3)
```

**设计原则**：
- 阴影颜色偏冷（略带蓝色），避免"脏灰"
- 使用双层阴影（Y轴偏移+模糊）营造立体感
- 阴影透明度控制在 2%-6%，极度克制

### 2.4 间距系统（8px 基准）

```
基础单位: 4px

间距阶梯:
- 1: 4px（图标与文字间距）
- 2: 8px（紧凑内边距）
- 3: 12px（标准组件内边距）
- 4: 16px（卡片内边距）
- 5: 20px（模块间距）
- 6: 24px（大模块间距）
- 8: 32px（页面级间距）
- 10: 40px（超大间距）
```

**核心规则**：相邻元素间距遵循 8px 倍数，保持节奏感。

### 2.5 排版系统

| 层级 | 字号 | 字重 | 行高 | 用途 |
|-----|-----|-----|-----|-----|
| H1 | 24px | 600 | 32px | 页面标题 |
| H2 | 18px | 600 | 26px | 模块标题 |
| H3 | 16px | 600 | 24px | 卡片标题 |
| Body | 14px | 400 | 22px | 正文内容 |
| Caption | 12px | 500 | 18px | 标签、辅助文字 |
| Tiny | 11px | 500 | 16px | 时间戳、极次要信息 |

## 3. 模块优化方案

### 3.1 左侧导航（Sidebar）

#### 现状问题
- 选中态使用纯色背景，过于突兀
- 分割线 1px solid，割裂感强
- Logo 区域与导航项缺乏层次

#### 优化方案

**整体风格**：
- 背景：#FAFAFA（极浅灰，与主内容区 #FFFFFF 形成微妙区分）
- 宽度：240px（当前 256px 略宽）

**Logo 区域**：
- 移除底部分割线，改用 24px 下边距留白
- Logo 图标使用 36×36px，圆角 10px
- 品牌名：16px/600，字色 Primary

**导航项**：
```
默认态：
- 背景：透明
- 文字：rgba(0,0,0,0.56)
- 图标：rgba(0,0,0,0.36)
- 内边距：10px 12px
- 圆角：10px
- 间距：4px（项与项之间）

悬停态：
- 背景：rgba(0,0,0,0.04)
- 文字：rgba(0,0,0,0.88)
- 过渡：all 200ms ease-out

选中态：
- 背景：rgba(124,126,246,0.08)（主色 8% 透明度）
- 文字：#7C7EF6（主色）
- 图标：#7C7EF6
- 左侧 3px 主色指示条（圆角 2px）
```

**分割线替代**：
- 移除显式分割线
- 使用 20px 间距 + 背景色差异实现视觉分组

### 3.2 中间对话区（Chat Area）

#### 现状问题
- 消息气泡边框明显，缺乏"气泡"感
- 用户与 Agent 消息视觉权重相同
- 工具调用卡片过于复杂，喧宾夺主

#### 优化方案

**消息气泡系统**：

```
用户消息：
- 背景：linear-gradient(135deg, #7C7EF6 0%, #6366F1 100%)
- 文字：#FFFFFF
- 圆角：16px（左上、左下、右下圆角，右上 4px）
- 阴影：0 2px 8px rgba(124,126,246,0.25)
- 最大宽度：75%
- 内边距：12px 16px

Agent 消息：
- 背景：#FFFFFF
- 边框：无（用阴影定义边界）
- 阴影：0 1px 3px rgba(0,0,0,0.04)
- 圆角：16px（右上、左下、右下圆角，左上 4px）
- 最大宽度：85%
- 内边距：12px 16px
```

**时间戳**：
- 字号：11px
- 颜色：rgba(0,0,0,0.22)
- 位置：气泡下方 4px，与气泡左/右对齐

**工具调用（极简版）**：
```
收起态：
- 背景：rgba(52,211,153,0.08)（Success 色 8%）
- 圆角：10px
- 高度：36px
- 左侧：绿色圆点指示器 6px
- 文字：工具名称，12px/500
- 右侧：ChevronDown 图标

展开态：
- 背景：#FFFFFF
- 边框：1px solid rgba(0,0,0,0.06)
- 圆角：10px
- 内边距：12px
- 参数/结果使用等宽字体，12px，灰色背景代码块
```

**错误提示**：
```
- 背景：rgba(248,113,113,0.08)（Error 色 8%）
- 圆角：10px
- 左侧：4px 红色指示条
- 图标：AlertCircle，16px，红色
- 文字：14px，rgba(0,0,0,0.72)
```

### 3.3 右侧详情面板（RightPanel）

#### 现状问题
- 状态卡片过于抢眼，喧宾夺主
- 工具网格使用边框，显得细碎
- 整体缺乏"面板"的统一感

#### 优化方案

**整体风格**：
- 背景：#FFFFFF（与主内容区统一，用间距分隔）
- 左边距：1px 极浅分割线 rgba(0,0,0,0.06)

**状态卡片（去色化）**：
```
- 背景：#FAFAFA（而非渐变）
- 圆角：16px
- 内边距：16px
- 状态指示：16px 圆形图标 + 状态文字
- 版本号：Caption 样式，灰色
- 移除"运行中"大字号，改为图标+文字组合

连接正常：绿色圆点 + "服务正常"
连接断开：红色圆点 + "连接中断"
```

**工具网格（无框化）**：
```
- 网格：2 列，间距 8px
- 工具项：
  - 背景：透明
  - 圆角：10px
  - 内边距：12px
  - 悬停：背景 rgba(0,0,0,0.04)
  - 图标：32px 圆形，主色背景，白色图标
  - 名称：12px/500，居中
```

**系统配置卡片**：
```
- 背景：#FAFAFA
- 圆角：12px
- 内边距：16px
- 行间距：12px
- 标签：Caption，灰色
- 值：Body，黑色
```

### 3.4 底部输入区（Input Area）

#### 现状问题
- 输入框与按钮分离，缺乏融合感
- 顶部边框明显，割裂对话流
- 整体过于"工具化"

#### 优化方案

**整体容器**：
```
- 背景：#FFFFFF
- 上边框：无（用 24px 上边距留白分隔）
- 内边距：16px 24px
```

**输入框（融合设计）**：
```
- 背景：#F5F5F5（浅灰，与白色页面形成对比）
- 圆角：20px（胶囊形）
- 高度：48px（单行）/ 自适应（多行）
- 内边距：12px 20px
- 边框：无
- 聚焦：背景 #FFFFFF，阴影 0 0 0 3px rgba(124,126,246,0.15)
- Placeholder：rgba(0,0,0,0.36)
```

**发送按钮（内嵌式）**：
```
- 位置：输入框右侧内部，距右边缘 6px
- 尺寸：36×36px
- 圆角：12px（与输入框圆角协调）
- 背景：#7C7EF6
- 图标：Send，18px，白色
- 悬停：背景 #6366F1，缩放 1.05
```

**快捷提示**：
```
- 位置：输入框下方 8px，左侧对齐
- 文字："Ctrl + Enter 发送"
- 样式：Caption，rgba(0,0,0,0.36)
```

## 4. 交互与动效规范

### 4.1 过渡曲线

```
标准过渡：200ms ease-out（按钮悬停、颜色变化）
弹性过渡：300ms cubic-bezier(0.34, 1.56, 0.64, 1)（卡片悬停、弹出）
平滑过渡：400ms cubic-bezier(0.4, 0, 0.2, 1)（页面切换、大元素）
快速反馈：100ms ease-out（按下态、小元素）
```

### 4.2 状态反馈

**悬停态（Hover）**：
- 背景色变化：透明度 +4%
- 轻微上浮：translateY(-1px)
- 阴影提升：从 shadow-sm 到 shadow-md

**按下态（Active）**：
- 缩放：scale(0.98)
- 背景：透明度 +2%

**聚焦态（Focus）**：
- 光环：0 0 0 3px rgba(124,126,246,0.15)
- 无 outline

**选中态（Selected）**：
- 背景：主色 8% 透明度
- 文字：主色

### 4.3 加载与流式输出

**思考中状态**：
```
- 显示："思考中..." + 跳动光点
- 光点：3 个，依次透明度变化
- 动画：每个点 0.6s 循环，相位差 0.2s
```

**流式输出**：
```
- 文字逐字显示（打字机效果）
- 光标：竖线，主色，闪烁 1s 循环
- 速度：20ms/字（可调节）
```

**骨架屏**：
```
- 背景：linear-gradient(90deg, #F5F5F5 0%, #FAFAFA 50%, #F5F5F5 100%)
- 动画：背景位置移动，1.5s 循环
- 圆角：与内容一致
```

## 5. 设计验收标准

### 5.1 视觉检查清单

- [ ] 界面中无 1px solid 硬边框（除极特殊情况）
- [ ] 所有圆角遵循 6/8/10/12/16/20px 阶梯
- [ ] 阴影使用双层，颜色偏冷，透明度 ≤6%
- [ ] 背景色阶过渡自然，无跳跃感
- [ ] 文字对比度符合 WCAG AA 标准
- [ ] 交互元素有明确的 Hover/Active/Focus 反馈

### 5.2 体验检查清单

- [ ] 悬停时元素有轻微上浮感
- [ ] 点击时有明确的按下反馈
- [ ] 加载状态不突兀，有呼吸感
- [ ] 错误提示温和，不刺眼
- [ ] 整体视觉焦点集中在对话内容

### 5.3 成功指标

| 指标 | 现状 | 目标 |
|-----|-----|-----|
| 界面边框数量 | 15+ | ≤3 |
| 阴影层级统一度 | 60% | 100% |
| 圆角一致性 | 70% | 100% |
| 用户主观精致感评分 | 5/10 | 8/10 |
